# "Interpretation Rules.txt"

# Interprétation de métrage automatique Adobe After Effects

# les lignes vierges sont autorisées et les lignes commençant par "#" sont des commentaires

# format de base d'une règle stricte : {exigences de concordance} = {interprétation définie}
# format de base d'une règle souple : {exigences de concordance} ~ {interprétation définie}

# + pour chaque règle concordante, l'interprétation est définie ; les caractères génériques ("*") sont autorisés
#	de part et d’autre ; la correspondance de caractère générique correspond à n’importe quel caractère, le jeu de caractères génériques ne définit rien
# + plusieurs règles peuvent concorder, les plus récentes remplaçant les plus anciennes
# + toutes les règles sont balayées dans l'ordre, une par ligne
# + les règles sont appliquées après lecture du format des pixels, des informations alpha et des informations d'identification de trame
#	informations lues à partir d’un fichier

# + les règles logicielles n’écraseront pas les valeurs explicitement spécifiées par le fichier
#	format ou défini par les règles précédentes

# largeur, hauteur, fréquence des images, type de fichier, codec =
#	format/nom des pixels, ordre de trame, cadence conforme, interpolation alpha, code de profil icc
#	ou description du profil (l’un ou l’autre), lumière linéaire

# la largeur et la hauteur sont des nombres entiers
# la cadence est un nombre décimal
# le fichier est du type quatre caractères entre guillemets (FourCC)
# le codec (applicable pour QT et AVI uniquement) est également du type FourCC

# pour rechercher le type de fichier et/ou le code de profil FourCC du codec pour un type particulier de métrage,
#	opt-clic (Mac) ou alt-clic (Windows) sur le métrage dans la fenêtre Projet ;
#	les codes FourCC s’affichent sur la dernière ligne de texte près de la vignette

# le format des pixels est un rapport d'entiers (ex. : 10/11)
#	éventuellement suivi d’un nom pour ce taux (ex. 10/11/"Mon aspect")
# Dans After Effects CS4 (version 9), nous avons modifié le format des pixels pour plusieurs formats communs :
#	D1/DV NTSC de 648/720 (0,9) à 10/11 (environ 0,9091)
#	D1/DV NTSC Grand écran de 6/5 (1,2) à 40/33 (environ 1,2121)
#	D1/DV PAL de 768/720 (1,07) à 768/702 (environ 1,0940)
#	D1/DV PAL Grand écran de 64/45 (1,42) à 1 024/702 (environ 1,4587)

# l'ordre de trame est un caractère unique, (F = image, U = trame supérieure ou L = trame inférieure)
# la cadence conforme est un nombre décimal
# interpolation alpha : (I = ignorer, S = directe, P = prémult./noir, W = prémult./blanc)
#	interpolation alpha ignorée pour le métrage sans couche alpha

# Le profil ICC est précisé par le code de profil (FourCC) ou par
#	 description du profil qui est une chaîne entre guillemets avec 255 caractères max. plus les guillemets

# Les codes de profil pris en charge sont les suivants :
#	- profil de l’espace de travail : "wRGB"
#	- profil incorporé : "embe"
#	- conserver les valeurs RVB : "pRGB" (revient à cocher la case Conserver les valeurs RVB dans la boîte de dialogue Interpréter le métrage)
#	- sRGB : "sRGB"
#	- RVB Adobe : "AS98"
#	- Profil de film Kodak 5218 : "ginp"
#	- SDTV NTSC : "r6nf" (primaires SMPTE RP 145 et TRC ITU-R Rec. 709. Le noir est codé à 0 et le blanc à 1.)
#	- SDTV NTSC 16-235 : "r6ns" (primaires SMPTE RP 145 et TRC ITU-R Rec. 709. Le noir est codé à 16/255 et le blanc à 235/255.)
#	- SDTV PAL : "r6pf" (primaires EBU Tech. 3213 et TRC ITU-R Rec. 709 =. Le noir est codé à 0 et le blanc à 1.)
#	- SDTV PAL 16-235 : "r6ps" (primaires EBU Tech. 3213 et TRC ITU-R Rec. 709. Le noir est codé à 16/255 et le blanc à 235/255.)
#	- HDTV (Rec. 709) : "r7hf" (primaires et TRC ITU-R Rec. 709. Le noir est codé à 0 et le blanc à 1.)
#	- HDTV (Rec. 709) 16-235 : "r7hs" (primaires et TRC ITU-R Rec. 709. Le noir est codé à 16/255 et le blanc à 235/255.)

# La description du profil peut être consultée dans l’interface utilisateur (infos de la vignette de la fenêtre Projet) ou dans
# utilitaire chromatique tel que ColorSync ou est défini par l’utilisateur lors de la création d’un profil personnalisé

# la lumière linéaire est un nombre entier (0 = Non, 1 = Oui, 2 = Oui pour les métrages 32 bpc)
