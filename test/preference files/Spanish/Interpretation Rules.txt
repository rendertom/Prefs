# "Interpretation Rules.txt"

# Interpretación automática del material de archivo de Adobe After Effects

# se permiten las líneas en blanco y las que comienzan con "#" son comentarios

# formato básico de una regla dura {requisitos de coincidencia} = {definir interpretación}
# formato básico de una regla blanda {requisitos de coincidencia} ~ {definir interpretación}

# +por cada regla que coincide, se define la interpretación; se permiten caracteres comodín(*)
#	en ambos lados; la coincidencia de los caracteres comodín no coincide con nada, la definición de los caracteres comodín no define nada
# + se permite que coincidan varias reglas, las coincidencias posteriores anulan a las anteriores
# + todas las reglas se escanean en orden, una regla por línea
# + las reglas se aplican después de la proporción de píxeles, de la información de alfa y del campo de etiqueta
#	la información se lee desde un archivo

# + las reglas blandas no anularán valores que se especifican explícitamente mediante el formato de archivo
#	formato o conjunto de reglas anteriores

# anchura, altura, velocidad de fotogramas, tipo de archivo, códec =
#	 proporción de aspecto de píxeles/nombre, orden de campo, conformar velocidad de fotogramas, interpretación alfa, código de perfil icc
#	o descripción del perfil (una o la otra), luz lineal

# la anchura y la altura son números enteros
# la velocidad de fotogramas es un número decimal
# el tipo de archivo tiene cuatro caracteres entre comillas (FourCC)
# el códec (aplicable sólo para QT y AVI ) también presenta código FourCC

# para averiguar el tipo de archivo y/o los cuatro caracteres (FourCC) del códec de un tipo específico de material de archivo,
#	presione la tecla Opción (Mac) o Alt (Windows) y haga clic en el material de archivo de la ventana de proyecto,
#	los códigos FourCC aparecerán en la última línea de texto de la miniatura

# la proporción de aspecto de píxeles es una proporción de entero (p. ej. 10/11)
#	seguida opcionalmente por un nombre para esta proporción (p.ej. 10/11/"Mi aspecto")
# En After Effects CS4 (v9) hemos modificado las proporciones de aspecto de píxeles de varios formatos usuales:
#	D1/DV NTSC De 648/720 (0,9) a 10,0/11,0 (0,9091 aprox.)
#	D1/DV NTSC Pantalla ancha de 6/5 (1,2) a 40/33 (1,2121 aprox.)
#	D1/DV PAL Desde 768/720 (1,07) a 768/702 (1,0940 aprox.)
#	D1/DV PAL Pantalla ancha de 64/45 (1,42) a 1024,0/702,0 (1,4587 aprox.)

# el orden de campo es sólo un carácter (F=fotograma, S=campo superior, I=campo inferior)
# la velocidad de fotogramas ajustada es un número decimal
# interpretación de alfa: (I=ignorar, R=recto, P=premul/negro, B=premul/blanco)
#	la interpretación de alfa se omite para el material de archivo sin canal alfa

# El perfil ICC se especifica mediante el código del perfil que tiene 4 caracteres o mediante
#	descripción del perfil que es una cadena entre comillas con 255 caracteres como máximo más las comillas

# Los códigos de perfil que se admiten son:
#	- perfil de espacio de trabajo: "wRGB"
#	- perfil incrustado: "embe"
#	- conservar RGB: "pRGB" (tiene la misma función que la casilla de verificación Conservar RGB en el cuadro de diálogo Interpretar material de archivo)
#	- sRGB: "sRGB"
#	- Adobe RGB: "AS98"
#	- Perfil de la película Kodak 5218: "ginp"
#	- SDTV NTSC: "r6nf" (145 primarios SMPTE RP y rectángulo ITU-R. 709 TRCs. El negro se codifica como 0 y el blanco como 1).
#	- SDTV NTSC 16-235: "r6ns" (145 primarios SMPTE RP y rectángulo ITU-R. 709 TRCs. El negro se codifica como 16/255 y el blanco como 235/255).
#	- SDTV PAL: "r6pf" (3213 primarios EBU Tech. y rectángulo ITU-R 709 TRCs. El negro se codifica como 0 y el blanco como 1).
#	- SDTV PAL 16-235: "r6ps" (3213 primarios EBU Tech. y rectángulo ITU-R 709 TRCs. El negro se codifica como 16/255 y el blanco como 235/255).
#	- HDTV (Rectángulo 709): "r7hf" (709 primarios de rectángulo ITU-R y TRCs. El negro se codifica como 0 y el blanco como 1).
#	- HDTV (Rectángulo 709) 16-235: "r7hs" (709 primarios de rectángulo ITU-R y TRCs. El negro se codifica como 16/255 y el blanco como 235/255).

# La descripción del perfil se puede buscar en la IU (información de miniatura de ventana del proyecto) o en
# la utilidad de color, como ColorSync o está definido por el usuario al crear un perfil personalizado

# luz lineal es un número entero (0=desactivado; 1=activado, 2=activado para material de archivo de 32 bpc)
